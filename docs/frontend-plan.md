# MS-TMDB 前端实施计划

更新时间：2026-02-21

当前进展：`frontend/` 脚手架已创建（Vue 3 + TypeScript + Vite + TailwindCSS + Router + API 基础封装）。

## 1. 目标与范围

1. 基于现有后端接口快速搭建可用前端站点。
2. 优先覆盖核心用户路径：浏览、搜索、详情查看、管理操作。
3. 保持可迭代结构，后续可扩展到更多 TMDB 子接口。

## 2. 技术方案

1. 框架：Vue 3 + TypeScript + Vite。
2. 路由：Vue Router。
3. 状态管理：Pinia。
4. 请求层：Axios（统一封装请求、错误处理、超时与重试策略）。
5. 样式方案：TailwindCSS（含响应式断点与主题变量）。
6. 质量保障：ESLint + Prettier + Vitest。

## 3. 页面规划（MVP）

1. 首页 `/`：展示热门电影、热门电视剧、趋势内容。
2. 搜索页 `/search`：支持 movie/tv/person/multi 搜索与分页。
3. 电影详情页 `/movie/:id`：展示详情、演职员、图片、视频等模块。
4. 电视剧详情页 `/tv/:id`：展示详情、季集入口、子资源模块。
5. 人物详情页 `/person/:id`：展示人物资料与作品信息。
6. 管理页 `/admin`：本地覆盖编辑、同步、清理、统计。
7. 错误页：404、请求失败态、空数据态。

## 4. API 对接规划

1. 代理接口：统一走 `/api/v3/*`。
2. 管理接口：统一走 `/api/admin/*`。
3. 按模块拆分 API 文件：`movie.ts`、`tv.ts`、`person.ts`、`search.ts`、`admin.ts`。
4. 封装通用参数：`language`、`page`、`region`、`append_to_response`。
5. 统一错误处理：弹窗提示 + 页面级降级文案。

## 5. 迭代排期（建议 3 周）

1. 第 1 周：项目脚手架、路由、布局、请求层、首页和搜索页。
2. 第 2 周：电影/电视剧/人物详情页，完成主要子接口展示。
3. 第 3 周：管理页、联调修复、测试补齐、部署打包。

## 6. 验收标准

1. 所有 MVP 页面可访问，移动端和桌面端均可正常使用。
2. 核心接口链路可用：首页、搜索、详情、管理操作。
3. 常见异常可处理：接口超时、上游失败、空结果、参数错误。
4. 基础质量检查通过：`lint`、`typecheck`、`unit test`。

## 7. 风险与应对

1. 上游 TMDB 不稳定：前端增加重试与错误兜底展示。
2. 返回字段不稳定：增加类型保护和容错渲染。
3. 管理操作误用：增加二次确认和操作日志提示。

## 8. 下一步

1. 确认是否使用 Vue 3 方案（默认本计划）。
2. 确认 TailwindCSS 视觉方向（简洁后台风或内容平台风）。
3. 按新目录结构落地：
   - `backend/`：后端服务代码
   - `frontend/`：前端代码（Vue + TailwindCSS）
   - `docs/`：项目文档
   - `docker/`：容器与部署文件
4. 确认后即可开始完善 `frontend/` 脚手架并按周目标落地。
